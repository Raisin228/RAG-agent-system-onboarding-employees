# Настройка рабочего окружения

У каждого разработчика в FinBridge есть персональный dev-сервер — выделенная виртуальная машина, полностью изолированная от машин коллег. На ней ты разворачиваешь проект, запускаешь сервисы, дебажишь и пушишь код. Локально на своём ноутбуке ты только пишешь код в IDE — всё остальное происходит на dev-сервере.

## Характеристики dev-сервера

| Параметр | Значение |
|---|---|
| ОС | Ubuntu 22.04.4 LTS (Jammy Jellyfish) |
| CPU | 8 vCPU (Intel Xeon Gold 6348) |
| RAM | 32 GB |
| Диск | 256 GB SSD |
| Сеть | внутренняя сеть FinBridge, 10 Gbit/s между серверами |
| Хостинг | on-premise кластер в дата-центре DataLine (Москва, М9) |

Машина создаётся автоматически в день твоего выхода. Имя сервера — `dev-<твой-логин>`, например `dev-ivan.petrov`.

## Шаг 1. Получить доступ к dev-серверу

1. Напиши в **#it-support**: «Прошу выдать доступ к dev-серверу для <имя.фамилия>».
2. В ответ придёт сообщение с данными:
   - IP-адрес (вида `10.8.10.xxx`)
   - Логин (совпадает с корпоративным: `имя.фамилия`)
   - Временный пароль для первого входа
3. При первом подключении система потребует сменить пароль.

Среднее время выдачи доступа — 1–2 часа в рабочее время.

**Важно:** dev-сервер доступен только через VPN. Если ещё не настроил — сначала пройди инструкцию в `vpn_setup.md`.

## Шаг 2. Настроить SSH-подключение

### macOS

1. Открой Terminal.

2. Сгенерируй SSH-ключ (если ещё нет):
```bash
ssh-keygen -t ed25519 -C "имя.фамилия@finbridge.io"
```
Ключ сохранится в `~/.ssh/id_ed25519`. Парольную фразу (passphrase) рекомендуется установить.

3. Скопируй публичный ключ на dev-сервер:
```bash
ssh-copy-id имя.фамилия@10.8.10.xxx
```

4. Добавь конфигурацию в `~/.ssh/config`:
```
Host finbridge-dev
    HostName 10.8.10.xxx
    User имя.фамилия
    IdentityFile ~/.ssh/id_ed25519
    ForwardAgent yes
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

5. Теперь подключение — одной командой:
```bash
ssh finbridge-dev
```

### Linux (Ubuntu / Fedora)

Процесс идентичен macOS. Единственное отличие — если ssh-agent не запускается автоматически, добавь в `~/.bashrc`:

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

### Подключение через VS Code (рекомендуемый способ)

Большинство разработчиков в FinBridge пишут код локально в VS Code, но исполняют его на dev-сервере. Для этого:

1. Установи расширение **Remote - SSH** (`ms-vscode-remote.remote-ssh`).
2. Нажми `Ctrl+Shift+P` → **Remote-SSH: Connect to Host** → выбери `finbridge-dev` (он подтянется из `~/.ssh/config`).
3. VS Code откроет новое окно, подключённое к dev-серверу. Терминал, файловая система, дебаггер — всё будет работать на удалённой машине.

Документация расширения: [https://code.visualstudio.com/docs/remote/ssh](https://code.visualstudio.com/docs/remote/ssh)

## Шаг 3. Удалённый рабочий стол (RDP)

Если нужен графический интерфейс (например, для тестирования UI в браузере на сервере), можно подключиться по RDP.

На dev-сервере уже установлен **xrdp** + **XFCE4** (лёгкий десктоп).

### Подключение с macOS

1. Скачай **Microsoft Remote Desktop** из App Store: [https://apps.apple.com/app/microsoft-remote-desktop/id1295203466](https://apps.apple.com/app/microsoft-remote-desktop/id1295203466)
2. Добавь новое подключение: IP — `10.8.10.xxx`, порт — `3389`, логин и пароль — твои от dev-сервера.

### Подключение с Linux

```bash
sudo apt install remmina remmina-plugin-rdp
```

Открой Remmina → новое подключение → RDP → IP `10.8.10.xxx:3389`.

### Подключение с Windows

Встроенный клиент: Win+R → `mstsc` → IP `10.8.10.xxx`.

## Обновление зависимостей

Образ dev-сервера обновляется командой Platform Engineering раз в квартал. О предстоящих обновлениях предупреждают в **#engineering** за неделю.

Для ручного обновления системных пакетов:

```bash
sudo apt update && sudo apt upgrade -y
```

Для обновления `fb-stand` и внутренних утилит:

```bash
pip install --upgrade fb-stand --index-url https://pypi.finbridge.io/simple
```

**Не обновляй** Docker и Go самостоятельно — это может сломать совместимость с CI/CD пайплайном. Если нужна другая версия — напиши в **#platform-support**.

## Клонирование проекта

После подключения к dev-серверу склонируй основной репозиторий:

```bash
git clone git@gitlab.finbridge.io:finbridge/finbridge-core.git
cd finbridge-core
make setup  # установит pre-commit хуки, скачает зависимости, соберёт базовые образы
```

`make setup` выполняется 10–15 минут при первом запуске — скачиваются Docker-образы (~8 GB).

После этого:

```bash
make run  # поднимет все сервисы в Docker Compose
```

Проверка: открой в браузере (через RDP или SSH-туннель) `http://localhost:8080` — должна загрузиться панель FinBridge Pay.

## SSH-туннель как альтернатива RDP

Если не хочешь подключаться по RDP только чтобы открыть браузер, можно пробросить порт через SSH:

```bash
ssh -L 8080:localhost:8080 finbridge-dev
```

После этого `http://localhost:8080` на твоём ноутбуке покажет интерфейс, запущенный на dev-сервере.
