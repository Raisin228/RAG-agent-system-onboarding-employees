# Тестовые стенды FinBridge

Тестовый стенд — это изолированная копия продакшн-окружения, в которой можно безопасно тестировать фичи, воспроизводить баги и проверять интеграции. Каждый стенд содержит полный набор сервисов: API Gateway, процессинг транзакций, панель аналитики, админку и базу данных с синтетическими данными.

## Виды стендов

В FinBridge есть два типа стендов:

**Глобальные стенды** — общие для всей компании, всегда запущены и доступны:

| Стенд | URL | Назначение | Ветка |
|---|---|---|---|
| **dev** | `https://dev.finbridge.io` | Основная разработка, сюда мержится `develop` | `develop` |
| **staging** | `https://staging.finbridge.io` | Предрелизное тестирование, копия прода | `release/*` |
| **qa** | `https://qa.finbridge.io` | Ручное и автоматизированное тестирование | `develop` |

**Персональные стенды** — разворачиваются разработчиком под конкретную задачу, живут 72 часа и автоматически удаляются.

## Как посмотреть список всех стендов

Актуальный список всех запущенных стендов (глобальных и персональных) доступен в дашборде:

**`https://stands.finbridge.io`**

Там отображается: название стенда, URL, привязанная ветка, кто создал, сколько времени до удаления и текущий статус (healthy / degraded / down). Доступ — через VPN, авторизация по корпоративному SSO.

## Персональные стенды

### Как создать

Персональный стенд разворачивается через CLI-утилиту `fb-stand`:

```bash
# Установка утилиты (один раз)
pip install fb-stand --index-url https://pypi.finbridge.io/simple

# Создать стенд из текущей ветки
fb-stand create --branch feature/PAY-1234-new-checkout

# Проверить статус
fb-stand status

# Удалить досрочно
fb-stand destroy
```

После создания (занимает 5–8 минут) утилита выдаст URL вида `https://<твой-логин>.stands.finbridge.io`.

### Привязка стенда к рабочей машине

Ключевая особенность нашей системы — **автоматическая привязка разработчика к персональному стенду**. Работает это так:

1. Когда ты создаёшь персональный стенд, система регистрирует твой корпоративный аккаунт как владельца.
2. Все коммиты, пушнутые в привязанную ветку, автоматически деплоятся на твой стенд в течение 2–3 минут (CI/CD пайплайн в GitLab).
3. Тебе не нужно вручную деплоить или перезапускать сервисы — просто пушишь код и обновляешь страницу в браузере.

Чтобы проверить привязку:

```bash
fb-stand whoami
# Вывод: stand=ivan.petrov.stands.finbridge.io, branch=feature/PAY-1234-new-checkout, owner=ivan.petrov
```

Если нужно перепривязать стенд к другой ветке:

```bash
fb-stand rebind --branch feature/PAY-5678-refund-flow
```

### Авторизация на персональном стенде

На свой стенд ты заходишь под своей обычной корпоративной учёткой (`имя.фамилия@finbridge.io`). Система автоматически определяет что ты — владелец стенда и даёт полный доступ ко всем разделам: транзакции, аналитика, админка, логи.

## Доступ к глобальным стендам

Глобальные стенды (dev, staging, qa) не привязаны к конкретному разработчику. Чтобы посмотреть что на них происходит — используй учётные записи суперпользователей. У каждой учётки свой уровень доступа, соответствующий конкретному разделу системы:

### Учётные записи суперпользователей

| Раздел | Логин | Пароль | Что видно |
|---|---|---|---|
| Панель транзакций | `su.transactions@test.finbridge.io` | `TxTest2024!` | Все транзакции, возвраты, споры, детализация платежей |
| Аналитика и отчёты | `su.analytics@test.finbridge.io` | `AnReport2024!` | Дашборды, воронки, когортный анализ, выгрузки |
| Административная панель | `su.admin@test.finbridge.io` | `AdmPanel2024!` | Управление мерчантами, тарифы, лимиты, настройки системы |
| Мерчант-кабинет | `su.merchant@test.finbridge.io` | `MrcTest2024!` | Вид из кабинета клиента: баланс, выплаты, интеграция |
| Поддержка | `su.support@test.finbridge.io` | `SupHelp2024!` | Тикеты, чаты с мерчантами, внутренние заметки |

**Важно:** эти учётки работают только на стендах dev / staging / qa. На продакшн они не существуют.

### Как работать с несколькими стендами одновременно

Часто нужно одновременно видеть свой персональный стенд и глобальный staging. Браузер не даст залогиниться под двумя аккаунтами на одном домене, поэтому используй **вкладки в режиме инкогнито**:

1. Обычное окно браузера → твой персональный стенд (`https://<логин>.stands.finbridge.io`) под своей учёткой.
2. Первое окно инкогнито → `https://staging.finbridge.io` под `su.transactions@test.finbridge.io`.
3. Второе окно инкогнито → `https://dev.finbridge.io` под `su.analytics@test.finbridge.io`.

В Chrome каждое новое инкогнито-окно (Ctrl+Shift+N) имеет изолированную сессию, поэтому конфликтов авторизации не будет.

**Совет:** в Firefox есть расширение **Multi-Account Containers**, которое позволяет держать разные сессии в обычных вкладках без инкогнито — удобнее для постоянной работы.

## Синтетические данные на стендах

Все стенды наполнены синтетическими данными: 50 000 тестовых мерчантов, 3 миллиона транзакций за последние 6 месяцев, 200 000 возвратов. Данные генерируются скриптом `fb-seed` и обновляются на глобальных стендах каждый понедельник в 03:00 МСК.

На персональном стенде данные копируются один раз при создании. Если нужно обновить:

```bash
fb-stand seed --fresh
```
