# Политика информационной безопасности FinBridge

FinBridge обрабатывает платёжные данные и подчиняется стандарту **PCI DSS Level 1**. Это значит что требования к безопасности у нас строже, чем в обычных IT-компаниях. Нарушение политики безопасности — основание для блокировки доступов и дисциплинарных мер вплоть до увольнения. Это не формальность — мы обрабатываем 12 миллионов транзакций в месяц и несём ответственность перед мерчантами и их клиентами.

Ответственный за информационную безопасность — Игорь Савченко, Security Officer: @igor.savchenko в Slack, security@finbridge.io.

## Требования к устройствам сотрудников

Каждое устройство, с которого ты подключаешься к ресурсам FinBridge (корпоративный ноутбук, личный ноутбук, даже телефон со Slack), должно соответствовать следующим требованиям:

### Обязательно

| Требование | macOS | Windows | Linux |
|---|---|---|---|
| Шифрование диска | FileVault включён | BitLocker включён | LUKS на корневом разделе |
| Антивирус | CrowdStrike Falcon (выдаётся IT-отделом) | CrowdStrike Falcon | CrowdStrike Falcon |
| Пароль на вход в систему | Не менее 12 символов, буквы + цифры + спецсимволы | Аналогично | Аналогично |
| Автоблокировка экрана | Не более 5 минут бездействия | Не более 5 минут | Не более 5 минут |
| Обновления ОС | Не старше 30 дней от последнего релиза | Аналогично | Аналогично |
| Firewall | Включён (системный) | Включён (Windows Defender Firewall) | ufw / iptables активен |

### Запрещено

- Хранить пароли, токены, SSH-ключи и сертификаты в открытом виде (текстовые файлы, заметки, мессенджеры). Используй менеджер паролей — у нас корпоративная лицензия **1Password Business**, доступ запроси в **#it-support**.
- Устанавливать пиратское ПО.
- Подключать к рабочей машине незашифрованные внешние носители (флешки, HDD).
- Передавать VPN-сертификаты, SSH-ключи или учётные данные другим сотрудникам.
- Использовать публичный Wi-Fi без VPN. Даже для Slack.
- Делать скриншоты страниц с платёжными данными клиентов.

## Проверка устройства на соответствие

Перед первым подключением к VPN и далее раз в месяц необходимо запустить скрипт проверки, который определяет — соответствует ли твоя машина корпоративным требованиям.

### Установка

```bash
# macOS / Linux
curl -sSL https://security.finbridge.io/check/install.sh | bash
```

Скрипт установит утилиту `fb-security-check` в `/usr/local/bin/`.

### Запуск проверки

```bash
fb-security-check --full
```

Пример вывода при успешной проверке:

```
[✓] Disk encryption: FileVault enabled
[✓] Antivirus: CrowdStrike Falcon 7.14.0 running
[✓] Screen lock: 3 min (max 5 min)
[✓] OS update: macOS 15.4.1 (latest: 15.4.1)
[✓] Firewall: enabled
[✓] 1Password: installed, vault synced
[✓] VPN certificates: valid, expires 2025-11-03

STATUS: COMPLIANT
Report saved to ~/.finbridge/security-report-2025-04-08.json
```

Пример вывода при проблемах:

```
[✓] Disk encryption: FileVault enabled
[✗] Antivirus: CrowdStrike Falcon NOT FOUND
[✓] Screen lock: 3 min (max 5 min)
[✗] OS update: macOS 14.2 (latest: 15.4.1) — OUTDATED
[✓] Firewall: enabled
[✓] 1Password: installed, vault synced
[✓] VPN certificates: valid, expires 2025-11-03

STATUS: NON-COMPLIANT (2 issues)
Fix the issues above and re-run the check.
Report saved to ~/.finbridge/security-report-2025-04-08.json
```

Отчёт автоматически отправляется в систему безопасности. Если статус `NON-COMPLIANT` держится более 7 дней — VPN-доступ блокируется автоматически до устранения проблем.

### Установка CrowdStrike Falcon

Если проверка показала что антивирус не установлен:

1. Запроси установочный пакет в **#it-support**: «Прошу выдать установщик CrowdStrike для <macOS/Windows/Linux>».
2. IT-отдел пришлёт ссылку на скачивание и Customer ID для регистрации агента.
3. После установки перезагрузи машину и повторно запусти `fb-security-check --full`.

## Двухфакторная аутентификация (2FA)

2FA обязательна для всех корпоративных сервисов:

| Сервис | Способ 2FA |
|---|---|
| GitLab | TOTP через приложение (Google Authenticator, Authy) |
| Jira / Confluence | SSO через корпоративный Okta → 2FA в Okta |
| Slack | SSO через Okta |
| VPN | Клиентский сертификат (заменяет 2FA) |
| AWS Console | Hardware-ключ YubiKey (выдаётся тем, кому нужен доступ) |

Настройка Okta и подключение 2FA — в документе `access_requests.md`.

## Безопасность кода и продукта

### Работа с секретами

- Все секреты (API-ключи, пароли БД, токены) хранятся в **HashiCorp Vault**: `https://vault.finbridge.io`.
- В код **никогда** не коммитятся секреты. На всех репозиториях настроен pre-commit хук `detect-secrets`, который блокирует коммит при обнаружении чего-то похожего на ключ или пароль.
- Если случайно закоммитил секрет — немедленно сообщи в **#security** и отзови скомпрометированный ключ. Не пытайся просто удалить коммит — он уже в истории.

### Код-ревью с точки зрения безопасности

Все MR в репозитории `finbridge-core` проходят автоматическую проверку:

- **SAST** (Static Application Security Testing) — сканер `semgrep` проверяет код на типичные уязвимости (SQL-инъекции, XSS, path traversal).
- **Dependency scan** — `trivy` проверяет зависимости на известные CVE.
- **Secret scan** — `detect-secrets` + `gitleaks`.

Если сканер нашёл проблему — MR нельзя смержить до исправления. Результаты сканирования видны во вкладке **Security** в GitLab MR.

### Работа с данными клиентов

- На dev и staging стендах используются **только синтетические данные**. Реальные данные клиентов никогда не копируются на тестовые окружения.
- Доступ к продакшн-базам данных есть только у on-call инженеров и DBA-команды. Если нужно посмотреть прод-данные для дебага — запроси read-only доступ через тикет в Jira (проект `SEC`, тип задачи «Запрос доступа»). Срок рассмотрения — 1 рабочий день.
- Логирование: в логах сервисов **запрещено** выводить номера карт, CVV, персональные данные. Логгер автоматически маскирует паттерны карт (`4111 **** **** 1234`), но если пишешь кастомный лог — проверь вручную.
