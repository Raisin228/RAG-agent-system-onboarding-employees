# Настройка корпоративного VPN

Все внутренние сервисы FinBridge (GitLab, Jira, Confluence, Grafana, staging-кластер, dev-серверы) доступны только через корпоративный VPN. Без активного VPN-соединения ты не сможешь открыть ни один внутренний ресурс — браузер покажет ошибку подключения, а SSH до dev-сервера не пройдёт.

## Шаг 1. Установить OpenVPN Connect

Скачай клиент OpenVPN Connect с официального сайта:

- **Windows**: [https://openvpn.net/client/client-connect-vpn-for-windows/](https://openvpn.net/client/client-connect-vpn-for-windows/)
- **macOS**: [https://openvpn.net/client-connect-vpn-for-mac-os/](https://openvpn.net/client-connect-vpn-for-mac-os/)
- **Linux (Ubuntu/Debian):**
  ```bash
  sudo apt update && sudo apt install openvpn
  ```

Версия клиента должна быть **3.4 или выше**. Более ранние версии не поддерживают формат сертификатов, используемый в FinBridge.

## Шаг 2. Запросить сертификаты

Для подключения тебе нужен персональный набор файлов:

- `finbridge-ca.crt` — корневой сертификат компании
- `<твой_логин>.crt` — твой персональный клиентский сертификат
- `<твой_логин>.key` — приватный ключ
- `finbridge.ovpn` — конфигурационный файл с настройками сервера

Чтобы получить их:

1. Напиши в Slack-канал **#it-support** сообщение: «Прошу выдать VPN-сертификаты для <имя.фамилия>».
2. IT-отдел сгенерирует сертификаты и отправит тебе архив `vpn-<твой_логин>.zip` в личные сообщения в Slack. Обычно это занимает до 2 часов в рабочее время.
3. Архив защищён паролем — пароль придёт отдельным сообщением.

**Важно:** приватный ключ (`*.key`) нельзя пересылать другим людям, выкладывать в репозитории или хранить в облачных хранилищах. Это нарушение `security_policy.md` и ведёт к блокировке доступа.

## Шаг 3. Импортировать профиль в OpenVPN

### Windows / macOS (OpenVPN Connect)

1. Открой OpenVPN Connect.
2. Нажми **Import Profile** → **File**.
3. Выбери файл `finbridge.ovpn` из распакованного архива.
4. Клиент автоматически подтянет сертификаты, если они лежат в той же папке что и `.ovpn`. Если нет — укажи пути вручную в настройках профиля.
5. Нажми **Connect**.

### Linux

```bash
sudo openvpn --config /path/to/finbridge.ovpn
```

Для автоматического запуска при старте системы:

```bash
sudo cp finbridge.ovpn /etc/openvpn/client/finbridge.conf
sudo cp finbridge-ca.crt <логин>.crt <логин>.key /etc/openvpn/client/
sudo systemctl enable openvpn-client@finbridge
sudo systemctl start openvpn-client@finbridge
```

## Шаг 4. Проверить подключение

После подключения проверь что VPN работает:

```bash
# Должен вернуть IP из диапазона 10.8.0.0/24
curl ifconfig.me

# Должен резолвиться и отвечать
ping gitlab.finbridge.io

# Должен открыться в браузере
# https://gitlab.finbridge.io
```

Если `ping` не проходит, а VPN показывает статус «Connected» — проверь что в системе не активен другой VPN-клиент или прокси. Два VPN одновременно работать не будут.

## Частые проблемы

| Проблема | Решение |
|---|---|
| «TLS handshake failed» | Устаревшая версия клиента. Обнови OpenVPN Connect до 3.4+. |
| «Certificate has expired» | Сертификаты действуют 12 месяцев. Запроси перевыпуск в **#it-support**. |
| Подключается, но внутренние сайты не открываются | Проверь DNS: в настройках сети DNS-сервер должен быть `10.8.0.1` (выдаётся VPN автоматически). Если стоит сторонний DNS (Google 8.8.8.8) — он не знает про внутренние домены. |
| Постоянные обрывы соединения | Попробуй переключить протокол с UDP на TCP в настройках профиля. TCP стабильнее на плохих сетях, но медленнее. |
| «Auth failed» при подключении | Проверь что файлы `.crt` и `.key` соответствуют твоему логину, а не чужому. |

## Когда нужно включать VPN

- **Всегда**, когда работаешь не из офиса. В офисной сети (Wi-Fi `FinBridge-Office` или проводное подключение) VPN не нужен — внутренние ресурсы доступны напрямую.
- Перед `git push` / `git pull` — GitLab недоступен без VPN.
- Перед подключением к dev-серверу по SSH.
- Перед открытием Grafana, Kibana и других инструментов мониторинга.

По любым проблемам с VPN — пиши в **#it-support**, среднее время ответа — 12 минут. Если проблема критическая и блокирует работу — позвони Сергею Новожилову напрямую: +7 (926) 555-12-34.
